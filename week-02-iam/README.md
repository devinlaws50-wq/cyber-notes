## 📌 Objective

Understand and apply AWS Identity and Access Management (IAM) concepts, focusing on least-privilege access, role assumption, and debugging permission errors in real-world cloud scenarios.

---

## 🛠️ Tools & Technologies

- Amazon Web Services (AWS)
- AWS IAM
- AWS STS (Security Token Service)
- AWS EC2
- AWS Management Console

---

## 🧠 Topics Covered

- IAM users vs roles
- Least privilege access
- AWS managed vs custom policies
- STS role assumption
- Debugging AccessDenied errors

---

## ⚙️ Hands-On Work

- Created EC2 read-only IAM role
- Assumed role via AWS Console
- Observed permission enforcement in EC2
- Resolved IAM trust and policy misconfigurations

---

## 🧠 Key Concepts Learned

- Cloud security failures are most often identity failures
- Least privilege limits the blast radius of compromised accounts
- IAM roles are preferred over long-lived user credentials
- Trust policies and permission policies must both be correct for role assumption to succeed

---

## ✅ Outcome

Successfully configured and validated IAM roles and policies in AWS, demonstrating how identity-based access control enforces least-privilege security in a cloud environment.
